# TrajFlow 数据格式文档

本文档说明 TrajFlow 项目从原始 DiDi 数据到最终生成轨迹的完整数据流。包含真实数据样例（来自 `data/toy_data`）、关键代码段、以及每一步的维度变化。

## 1. 原始数据格式（DiDi GPS 轨迹）

### 来源

项目使用 **DiDi 滴滴出行 GPS 轨迹数据集**。原始数据是 CSV 文件，每一行是一个 GPS 点，多行属于同一趟出行（按用户ID、时间分组）。

### 字段说明

原始 DiDi CSV 中每行包含以下关键字段（见 `PrepareDataset.py` 第 85-147 行 `getCondTraj` 函数）：

```python

    lambda row: ju.to_meshcode(row['lat'], row['lon'], MESH_DEGREE), axis=1

```

| 字段名 | 类型 | 含义 | 示例 |
|--------|------|------|------|
| `lat` | float | 纬度 | `30.572815` |
| `lon` | float | 经度 | `104.066800` |
| `time` | str | 时间戳，格式 `YYYY-MM-DD HH:MM:SS` | `"2023-04-01 08:15:30"` |

### 4.2 轨迹标准化（norm1by1=True 时）

当 `config['data']['norm1by1'] == True` 时，对每条轨迹独立做质心居中和缩放（`dataset.py` 第298-331行）：

```python
def standardize_trajectories_preserve_aspect(self):
    standardized_segments = []
    for traj in self.traj_segments:
        # Step 1: 以轨迹质心为原点
        mean = traj.mean(axis=0)          # (2,) 纬度/经度各自的均值
        centered_traj = traj - mean

        # Step 2: 计算整体标准差（保持宽高比）
        std = np.sqrt(np.mean(np.sum(centered_traj ** 2, axis=1)))  # 标量

        # Step 3: 缩放
        if std < 1e-8:
            std = 1.0
        standardized_traj = centered_traj / std
        standardized_segments.append(standardized_traj)

    self.traj_segments = standardized_segments
```

**效果：所有轨迹归一化到 均值≈0、标准差≈1、坐标原点=轨迹质心。**

### 5.2 Flow Matching 训练步骤

```python
# trainer.py 第 192-248 行
def _train_step_flow_matching(self, batch):
    x_1, condition = batch
    x_1 = x_1.to(self.device)
    condition = condition.to(self.device)

    # 从标准正态分布采样噪声（x_0）
    x_0 = torch.randn_like(x_1).to(self.device)

    # 采样时间步 t ∈ [0, 1]
    t = torch.rand(batch_size).to(self.device)

    # 在 x_0 和 x_1 之间线性插值: x_t = (1-t)*x_0 + t*x_1
    path_sample = self.path.sample(t=t, x_0=x_0, x_1=x_1)

    # 真实速度: dx_t = x_1 - x_0（Conditional OT 的目标速度）
    # 模型预测速度: pred_v = model(x_t, t, condition)
    pred_v = self.model(path_sample.x_t, path_sample.t, condition)

    # MSE 损失: ||pred_v - (x_1 - x_0)||^2
    loss = torch.pow(pred_v - path_sample.dx_t, 2).mean()

    loss.backward()
    self.optimizer.step()
```

**训练输入/输出总结：**

```
x_0 (噪声):     [batch, 10, 2]     ~ N(0, I) 标准正态分布
t:              [batch]            ∈ [0, 1] 随机时间步
x_t:            [batch, 10, 2]    插值后的中间状态
condition:      [batch, 9]         9个条件字段
目标速度:       [batch, 10, 2]    x_1 - x_0（条件OT路径的切向量）
模型预测:       [batch, 10, 2]    速度场预测
```

### 5.3 条件编码器 WideAndDeep

模型 `ConditionalVelocityModel` 内部使用 WideAndDeep 将 9 维条件向量编码为嵌入向量（`networks.py` 第 680-777 行）：

```python
def forward(self, attr):
    # attr shape: [batch, 9]
    # 列 1-5: 连续属性（total_dis, total_time, total_len, avg_dis, avg_speed）
    continuous_attrs = attr[:, 1:6]
    wide_out = self.wide_fc(continuous_attrs)           # [batch, 128]

    # 列 0: 出发时间（288桶分类嵌入）
    depature = attr[:, 0].long()
    depature_embed = self.depature_embedding(depature)  # [batch, 256]

    # 列 6: 起点区域ID（Embedding 层）
    sid = attr[:, 6].long() + 0  # +encoding_bias=0
    sid_embed = self.sid_embedding(sid)                  # [batch, 256]

    # 列 7: 终点区域ID（Embedding 层）
    eid = attr[:, 7].long() + 0
    eid_embed = self.eid_embedding(eid)                  # [batch, 256]

    # 列 8: 交通方式（Embedding 层，transportation_mode=True）
    tmode = attr[:, 8].long()
    tmode_embed = self.tmode_embedding(tmode)           # [batch, 256]

    # Deep part: 拼接 + MLP
    categorical_embed = torch.cat(
        [depature_embed, sid_embed, eid_embed, tmode_embed], dim=1
    )  # [batch, 1024]
    deep_out = F.relu(self.deep_fc1(categorical_embed)) # [batch, 128]
    deep_out = self.deep_fc2(deep_out)                  # [batch, 128]

    # Wide + Deep 融合
    combined_embed = wide_out + deep_out  # [batch, 128]
    return combined_embed
```

然后该嵌入与时间步 embedding 拼接后输入 UNet：


## 6. 推理生成流程

### 6.1 从噪声生成轨迹

```python
# inference.py 第 263-339 行
def _sample_flow_matching(self, n_samples, n_steps=10, method='em', condition=None):
    # 从标准正态分布采样初始噪声
    x_init = torch.randn((n_samples, self.M, 2), dtype=torch.float32, device=self.device)
    # M = 10（parametrized_M=10）

    # Euler-Maruyama 数值积分
    time_grid = torch.linspace(0, 1, n_steps).to(self.device)
    step_size = 1.0 / (n_steps - 1)
    x_t = x_init.clone()

    for t_idx in range(n_steps - 1):
        t = time_grid[t_idx] * torch.ones(n_samples, 1, device=self.device)
        # 模型预测速度
        v_t = sampling_model(x_t, t, c=condition)
        # 欧拉更新: x_{t+1} = x_t + v_t * step_size
        x_t = x_t + v_t * step_size

    return [x_init, x_t]  # [初始噪声, 最终轨迹]
```

### 6.3 反标准化（标准化坐标 → 真实经纬度）

当 `norm1by1=True` 时，需要将标准化坐标反变换回真实经纬度（`inference.py` 第 759-863行 `MixStrategy` 方法）：

```python
# inference.py 第 759-863 行（MixStrategy 策略）
for i in range(batch_size):
    traj_norm = trajectories_reshaped[i]  # 标准化后的轨迹 [120, 2]

    origin_norm = traj_norm[0]      # 生成轨迹的标准化起点
    dest_norm = traj_norm[-1]      # 生成轨迹的标准化终点

    origin_target = condition_sample_lonlat[i, 0]  # 真实目标起点 [lat, lon]
    dest_target = condition_sample_lonlat[i, 1]    # 真实目标终点 [lat, lon]

    # 计算相似变换参数
    V_norm = dest_norm - origin_norm
    V_target = dest_target - origin_target
    len_norm = np.linalg.norm(V_norm)
    len_target = np.linalg.norm(V_target)
    scale_s = len_target / len_norm  # 缩放因子

    angle_norm = np.arctan2(V_norm[1], V_norm[0])
    angle_target = np.arctan2(V_target[1], V_target[0])
    theta = angle_target - angle_norm
    cos_theta, sin_theta = np.cos(theta), np.sin(theta)

    # 相似变换: 缩放 + 旋转 + 平移
    for j in range(traj_length):
        p_centered = traj_norm[j] - origin_norm
        p_scaled = p_centered * scale_s
        p_rotated_x = p_scaled[0] * cos_theta - p_scaled[1] * sin_theta
        p_rotated_y = p_scaled[0] * sin_theta + p_scaled[1] * cos_theta
        denorm_trajectories[i, j, 0] = p_rotated_x + origin_target[0]
        denorm_trajectories[i, j, 1] = p_rotated_y + origin_target[1]

    # 端点校正（消除浮点误差）
    start_error = origin_target - denorm_trajectories[i, 0]
    end_error = dest_target - denorm_trajectories[i, -1]
    w = np.linspace(1, 0, traj_length)
    correction = w[:, np.newaxis] * start_error + (1 - w)[:, np.newaxis] * end_error
    denorm_trajectories[i] = denorm_trajectories[i] + correction
```

**效果：生成轨迹的起点/终点精确匹配条件中的 OD 坐标，整体形状按比例、旋转、平移变换。**

```python
            "latitude": traj[j, 0],                # 生成轨迹纬度
            "longitude": traj[j, 1],                # 生成轨迹经度
        })
```

## 8. 完整数据流总结

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          原始 DiDi GPS 轨迹 CSV                             │
│  lat, lon, time, trans_mode                                                │
│  例: 30.572815, 104.066800, 2023-04-01 08:15:30, CAR                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  getCondTraj() 计算统计量 + jismesh 编码
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                     预处理后 pickle 文件                                     │
│  traj_segments.pkl     → [N, 120, 2]   每条轨迹120个GPS点                    │
│  conditions.pkl       → [N, 9]        9个条件字段（部分标准化）               │
│  mesh_mapping_dict.pkl → {geohash_int: id}  区域编码映射                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  DataLoader 加载 + standardize（norm1by1=True）
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Dataset.__getitem__ 输出                                 │
│  x_1: [10, 2]   (parametrized=True 时) 或 [120, 2] (parametrized=False)     │
│  condition: [9]  出发时间、总距离、总时间、...、OD区域ID、交通方式             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  Flow Matching 训练 (Conditional OT)
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                           模型学习                                          │
│  x_0 ~ N(0,I) ──线性插值──► x_t = (1-t)*x_0 + t*x_1                        │
│  目标: 预测速度 v_t = x_1 - x_0                                            │
│  Loss: MSE(v_pred, x_1 - x_0)                                              │
│                                                                             │
│  条件编码 WideAndDeep:                                                      │
│    连续属性(5维) → Linear → [batch, 128]                                    │
│    出发时间(288类) → Embedding(288,256)                                     │
│    起点区域 → Embedding(num_locations,256)                                  │
│    终点区域 → Embedding(num_locations,256)                                  │
│    交通方式(5类) → Embedding(5,256)                                        │
│    拼接 → MLP → [batch, 128]                                               │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  推理: 从噪声生成
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                     生成结果 [10, 2]                                        │
│  x_init ~ N(0,I) → Euler积分 10步 → x_final (10个关键点)                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  para2point (rdp_k 插值还原)
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                     还原后轨迹 [120, 2]                                     │
│  10个关键点 → 弧长参数化 → 线性插值 → 120个点                                │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  MixStrategy 反标准化（norm1by1=True）
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                     最终经纬度轨迹 [120, 2]                                  │
│  相似变换(缩放+旋转+平移) + 端点校正                                          │
│  使起点/终点精确匹配条件中的 OD 网格坐标                                      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  save_trajectories_to_csv()
                                    │
┌─────────────────────────────────────────────────────────────────────────────┐
│                        输出 CSV                                             │
│  uid, departure_time, total_dis, ..., latitude, longitude                   │
│  每条轨迹120行，总共 N*120 行                                                │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 9. 维度变化总览

| 阶段 | 张量形状 | 说明 |
|------|----------|------|
| 原始 CSV | `[N_raw, 1]` per point | 每行一个GPS点 |
| traj_segments.pkl | `[N, 120, 2]` | 每条120点，每点[lat,lon] |
| 标准化后 | `[N, 120, 2]` | 均值0，std≈1，原点=质心 |
| 参数化后（训练） | `[N, 10, 2]` | RDP-K 压缩为10个关键点 |
| 条件向量 | `[N, 9]` | 出发时间+5统计量+OD+交通方式 |
| 模型输入 x_t | `[B, 10, 2]` | Batch中一条轨迹的中间状态 |
| 模型输入 t | `[B]` | Batch中每条的时间步 |
| 模型输入 condition | `[B, 9]` | Batch中每条的条件 |
| 模型输出 v_t | `[B, 10, 2]` | 预测的速度场 |
| 推理生成 | `[B, 10, 2]` | 10个生成的关键点 |
| 插值还原 | `[B, 120, 2]` | 还原为120个点 |
| 反标准化 | `[B, 120, 2]` | 真实经纬度坐标 |
| 输出 CSV | `N*120 rows` | 每行一个点，带完整条件 |

## 10. 配置对照表

不同 `config.yaml` 组合决定数据走哪条路径：

| 配置项 | 默认值(Toy) | 影响 |
|--------|------------|------|
| `data.parametrized` | `true` | `true`: 模型输入 [10,2] / `false`: [120,2] |
| `data.parametrized_method` | `rdp_k` | RDP关键点压缩方法 |
| `data.parametrized_M` | `10` | 关键点数量 |
| `data.norm1by1` | `true` | `true`: 质心居中+缩放 / `false`: 全局z-score |
| `data.od_finer` | `false` | `true`: 额外4维网格内偏移参数 |
| `data.geohash` | `false` | `true`: 用geohash编码OD / `false`: jismesh |
| `condition.enabled` | `true` | 是否有条件输入 |
| `condition.condition_type` | `full` | `full`: 全部9维 / `od`: 仅OD 2维 |
| `condition.transportation_mode` | `true` | 是否包含交通方式embedding |
| `flow_matching.flow_type` | `standard` | Conditional OT 路径类型 |
